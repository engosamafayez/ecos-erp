# ECOS ERP — Production Deployment

## Architecture

```
Build pipeline
  Stage 1  composer     composer install --no-dev --no-scripts
                        composer dump-autoload --no-dev --optimize
  Stage 2  frontend     npm ci && npm run build → /backend/public/app
  Stage 3  app          php:8.4-fpm-bookworm + vendor + SPA + metadata
                        → image: ecos-erp/app:latest
  Stage 4  nginx        nginx:1.27-alpine + SPA + build-info
                        → image: ecos-erp/nginx:latest

Runtime
  ecos-app    ← ecos-erp/app    PHP-FPM + Supervisor (queue + scheduler)
  ecos-nginx  ← ecos-erp/nginx  nginx static server + PHP-FPM proxy
  ecos-mysql  ← mysql:8.4
  ecos-redis  ← redis:7-alpine
  ecos-mailpit ← axllent/mailpit (replace with real MTA before go-live)

Named volumes (runtime-mutable data only)
  app-storage  → /var/www/html/storage          uploads, sessions, logs
  app-cache    → /var/www/html/bootstrap/cache  artisan optimize output
  mysql-data   → /var/lib/mysql
  redis-data   → /data
```

---

## Why Source Bind-Mounts Are Forbidden in Production

A Docker bind-mount at `/var/www/html` **shadows** everything the image built
into that path — including `vendor/`, `public/app/`, and compiled views.

```
✗ WRONG (old approach):
  docker build → copies public/app into image
  docker compose up → mounts ./backend:/var/www/html
  Result: public/app, vendor, and all image artefacts are shadowed → 404 / 500

✓ CORRECT (current approach):
  docker build → image is self-contained (app + vendor + SPA + metadata)
  docker compose up → only mounts storage/ and bootstrap/cache/ (runtime data)
  Result: image serves exactly what was built and tested
```

---

## Why Bootstrap Cache Must Never Enter the Image

`backend/bootstrap/cache/packages.php` is generated by `artisan package:discover`
and lists all discovered service providers. If this file was created locally
(during development), it may include **dev-only providers** such as `laravel/pail`
that are not installed in production (`--no-dev`). Copying this file into the
image causes fatal class-not-found errors at startup.

**Fix applied:**
- `.dockerignore` excludes `backend/bootstrap/cache/*.php` from the build context.
- `entrypoint.sh` runs `artisan package:discover` at runtime (after
  `composer dump-autoload --no-dev`) to regenerate a clean `packages.php`
  without dev providers.
- Generated cache files go into the `app-cache` named volume, never into git
  or the image.

## Bootstrap Sequence (entrypoint.sh)

On every container start, in order:

```
1. Wait for MySQL (polls mysqladmin ping, 120 s timeout)
2. mkdir -p storage/{app/public,framework/{cache/data,sessions,views,testing},logs}
3. chown + chmod storage/ and bootstrap/cache/
4. php artisan storage:link          (skipped if symlink already exists)
5. php artisan key:generate          (skipped if APP_KEY already set)
6. composer dump-autoload --no-dev --optimize
7. php artisan package:discover      (regenerates packages.php without dev pkgs)
8. php artisan migrate --force       (idempotent: "Nothing to migrate" = exit 0)
9. php artisan optimize              (generates config/routes/views cache)
10. php artisan db:seed --class=AdminUserSeeder --force
11. exec supervisord                 (PHP-FPM + queue:work + schedule:work)
```

**Why dump-autoload runs at startup:** The vendor autoloader was built in
Stage 1 against the full application source. Running `dump-autoload --no-dev`
at runtime is a safety net that ensures the classmap is consistent with the
`--no-dev` vendor tree and provides the correct map for `package:discover`.

## Deployment Process

### Prerequisites

- Docker Engine 24+ with Compose V2
- Git repository cloned at `/opt/ecos-erp`
- `backend/.env` with `APP_DEBUG=false` and production credentials
- `docker-compose.override.yml` must NOT exist on the server

### First-time setup

```bash
cd /opt/ecos-erp
cp backend/.env.example backend/.env
nano backend/.env          # set APP_KEY, DB_*, MAIL_*, QUEUE_CONNECTION=redis

chmod +x deploy.sh
./deploy.sh --migrate      # first run always needs --migrate to create tables
```

### Regular deployment (code change, no schema change)

```bash
cd /opt/ecos-erp
./deploy.sh
```

### Deployment with schema change

```bash
cd /opt/ecos-erp
./deploy.sh --migrate
```

The `--migrate` flag is an **explicit operator decision**. Never run migrations
automatically — always confirm the migration is safe before passing this flag.

### What deploy.sh does

```
1. Validate: backend/.env exists, APP_DEBUG is false, override file absent
2. Tag current images as :rollback (for instant revert if needed)
3. git pull origin main
4. docker compose -f docker-compose.yml build --pull \
       --build-arg GIT_SHA=<sha> \
       --build-arg APP_VERSION=<version> \
       --build-arg BUILD_TIME=<utc-timestamp>
5. docker compose -f docker-compose.yml up -d --remove-orphans
6. Wait: ecos-app healthcheck → healthy (200 s timeout)
7. [--migrate] php artisan migrate --force
8. Verify: GET /healthz /build-info /app/ /api/health → all HTTP 200
9. Verify: index.html and build-info files present in both containers
10. docker image prune -f
```

If any step fails after containers were updated, the ERR trap automatically
restores the `:rollback` images and restarts.

---

## Rollback Procedure

### Automatic (via deploy.sh ERR trap)

If `deploy.sh` fails after containers were updated, rollback is automatic:
the script restores `:rollback` image tags and restarts.

### Manual instant rollback (no rebuild)

```bash
# Restore previous image tags
docker tag ecos-erp/app:rollback   ecos-erp/app:latest
docker tag ecos-erp/nginx:rollback ecos-erp/nginx:latest

# Restart from previous images (no rebuild)
docker compose -f docker-compose.yml up -d --no-build

# Verify
curl http://localhost/build-info
```

> If the failed deploy included `--migrate`, restore a database backup
> **before** restarting the previous image. Old code + new schema = undefined behaviour.

### Git revert + redeploy (schema-safe)

```bash
git revert HEAD
git push origin main
./deploy.sh              # add --migrate only if the revert includes a down migration
```

## Health Endpoints

| Endpoint | Source | Purpose |
|----------|--------|---------|
| `GET /healthz` | nginx static return | nginx alive check |
| `GET /build-info` | static JSON file in image | version / commit / build time |
| `GET /api/health` | nginx static return | infrastructure smoke test |
| `GET /app/` | nginx static file | SPA reachable |

```bash
# Verify which version is running
curl http://localhost/build-info
# → {"version":"v1.0.0","commit":"abc1234","built_at":"2026-06-28T10:00:00Z","environment":"production"}

# Human-readable from inside the container
docker exec ecos-app cat /var/www/html/.build-info

# Image label (no running container required)
docker inspect ecos-erp/app --format \
    '{{index .Config.Labels "org.opencontainers.image.revision"}}'
```

## Development Workflow

On a developer machine `docker-compose.override.yml` is auto-merged.
It restores source bind-mounts and replaces the custom nginx image with
the stock image for fast iteration:

```bash
# Dev (auto-merges override — source bind-mounts active)
docker compose up

# Production (explicit file — override excluded)
docker compose -f docker-compose.yml up
```

`docker-compose.override.yml` **must never be present on the production server.**
`deploy.sh` will abort if it detects the file.

## Troubleshooting

### 502 Bad Gateway

```bash
# Is PHP-FPM healthy?
docker inspect --format '{{json .State.Health}}' ecos-app

# FPM ping (manual)
docker exec ecos-app bash -c \
  'SCRIPT_NAME=/fpm-ping SCRIPT_FILENAME=/fpm-ping REQUEST_METHOD=GET \
   cgi-fcgi -bind -connect 127.0.0.1:9000'

# Logs
docker logs ecos-app --tail 50
```

### Laravel Pail / missing provider errors

This indicates a stale `bootstrap/cache/packages.php` was baked into the image
or the app-cache volume has a stale entry from a dev build.

```bash
# Wipe the app-cache volume and restart (will regenerate clean cache)
docker compose -f docker-compose.yml down
docker volume rm ecos-erp_app-cache
docker compose -f docker-compose.yml up -d
```

### GET /build-info returns 404

The `public/build-info` file was not copied into the nginx image (Stage 4).

```bash
docker exec ecos-nginx ls -la /var/www/html/public/build-info
docker compose -f docker-compose.yml build --no-cache
```

### Migration failures

```bash
docker compose -f docker-compose.yml exec app php artisan migrate:status
docker compose -f docker-compose.yml exec app php artisan migrate --force --verbose
```

### Storage permission errors

```bash
docker exec ecos-app bash -c \
  'chown -R www-data:www-data /var/www/html/storage /var/www/html/bootstrap/cache'
```

## Common Operational Commands

```bash
# Container status
docker compose -f docker-compose.yml ps

# Tail all logs
docker compose -f docker-compose.yml logs -f

# Shell into app container
docker compose -f docker-compose.yml exec app bash

# Any artisan command
docker compose -f docker-compose.yml exec app php artisan <command>

# Wipe Laravel caches (forces regeneration on next request)
docker compose -f docker-compose.yml exec app php artisan optimize:clear

# Queue monitor
docker compose -f docker-compose.yml exec app php artisan queue:monitor redis

# Force-restart a service
docker compose -f docker-compose.yml restart nginx
```

## File Layout (production server)

```
/opt/ecos-erp/
├── backend/
│   ├── .env                       ← production secrets (APP_DEBUG=false)
│   └── ...                        ← source; NOT mounted in production
├── frontend/                      ← source; NOT mounted in production
├── docker/
│   ├── php/
│   │   ├── Dockerfile             ← 4 stages: composer/frontend/app/nginx
│   │   ├── entrypoint.sh          ← runtime init (migrations, optimize, etc.)
│   │   ├── php.ini
│   │   ├── www.conf
│   │   └── supervisord.conf
│   ├── nginx/
│   │   └── default.conf           ← bind-mounted (tunable without rebuild)
│   └── mysql/
│       └── my.cnf
├── .dockerignore                  ← excludes bootstrap/cache, vendor, .env
├── docker-compose.yml             ← production (no source mounts)
│                                     DO NOT add override.yml on this server
└── deploy.sh                      ← single command for all deployments
```

## Known Limitations and Technical Debt

| Item | Status | Next step |
|------|--------|-----------|
| TLS / HTTPS | Not configured | Add TLS termination (Caddy / Traefik / nginx + certbot) |
| `GET /api/health` | nginx stub (no PHP) | Add real health route in `routes/api.php` |
| User-uploaded file serving | Via PHP-FPM (not nginx direct) | Mount `app-storage` read-only in nginx service |
| Mailpit in production | Stub only | Replace with SES / Mailgun / Postfix |
| Image registry | Local only | Push to registry for distributed rollback |
| Secrets in `.env` file | Plain file on disk | Migrate to Docker Secrets or vault |
