# ADR-011 V2.3 — Organization OS: Foundation Architecture

**ADR Number:** ADR-011  
**Version:** 2.3  
**Status:** APPROVED FOR IMPLEMENTATION  
**Date:** 2026-07-05  
**Supersedes:** ADR-011 V2.2 (Business Account as parent of Sales Channels)  
**See also:** ADR-012 (Marketing Assets Architecture)  
**CTO Sign-Off:** Approved — definitive organizational architecture for ECOS ERP  

---

## 1. Decision

Organization OS is the Foundation Module of ECOS ERP. It is the single source of truth for the enterprise organizational structure. No other module may redefine Company, Brand, Integration Account, Marketing Account, Sales Channel, Warehouse, Team, or User.

---

## 2. Enterprise Hierarchy

```
Company (COM-000001)
│
├── Brands (BRD-000001)
│      │
│      ├── Sales Channels (CH-000001)         ← Commerce Execution
│      │        │  Orders · Products · Pricing · Inventory · Analytics
│      │        └── (Commerce OS manages publishing and sync)
│      │
│      ├── Marketing Accounts (MKT-000001)    ← Marketing Infrastructure
│      │        │  Meta · Google · TikTok · Snapchat · LinkedIn
│      │        └── Marketing Assets
│      │                 Facebook Pages · Ad Accounts · Pixels · Catalogs
│      │                 └── (Future) Campaigns → Attribution → Orders
│      │
│      ├── Integration Accounts (IA-000001)   ← Technical Integration
│      │        │  OAuth · API Keys · Webhooks · Sync Settings
│      │        │  [formerly "Business Account" — table preserved as business_accounts]
│      │        └── (Technical credentials only — not parent of Sales Channels)
│      │
│      └── Brand Settings
│
├── Warehouses (WH-000001)
│
├── Teams (TM-000001)
│
├── Users
│
├── Roles & Permissions
│
└── Organization Settings
```

**Key change from V2.2:** Sales Channels now belong directly to Brand. Integration Accounts (formerly Business Accounts) are technical credential stores only — not the commerce hierarchy parent. Marketing Accounts are a new aggregate for marketing platform management.

---

## 3. Entity Definitions

### 3.1 Integration Account *(formerly Business Account)*

Integration Account represents a technical integration with an external platform. It owns credentials and sync configuration only. It does not own Sales Channels or marketing assets.

**Owns:**
- OAuth Tokens
- API Credentials (keys, secrets)
- Webhooks
- Sync Settings
- External Metadata

**Does NOT own:**
- Sales Channels (now owned directly by Brand)
- Marketing Assets (owned by Marketing Account)

**Platform examples:**
| Platform | Integration Account |
|---|---|
| WooCommerce | WooCommerce Store credentials |
| Shopify | Shopify Partner credentials |
| Amazon | Amazon Seller Central API |
| TikTok | TikTok open API credentials |
| Noon | Noon Seller API |

**Migration note:** Table remains `business_accounts` for Phase 1. Architectural term is "Integration Account". Code prefix target: `IA-{seq}`. Current deployed prefix: `BA-{seq}`.

### 3.2 Marketing Account

Marketing Account represents a marketing platform account owned by a Brand. It is the root for all marketing infrastructure — ad accounts, pixels, pages, catalogs, and future campaigns.

**Owns:**
- OAuth Tokens (marketing platform)
- API Credentials
- Webhooks (platform events)
- Sync Settings
- Marketing Assets (see §3.3)

**Platform examples:**
| Platform | Marketing Account |
|---|---|
| Meta | Meta Business Manager |
| Google | Google Merchant Center + Ads |
| TikTok | TikTok Business Center |
| Snapchat | Snapchat Business |
| LinkedIn | LinkedIn Campaign Manager |

**Code pattern:** `MKT-{6-digit seq per brand}` → `MKT-000001`

### 3.3 Marketing Assets

Marketing Assets are sub-entities of a Marketing Account. Each represents a specific platform object used for advertising, tracking, or content distribution.

**By provider:**

| Asset Type | Provider |
|---|---|
| `facebook_page` | Meta |
| `instagram_account` | Meta |
| `meta_ad_account` | Meta |
| `meta_pixel` | Meta |
| `meta_conversions_api` | Meta |
| `meta_catalog` | Meta |
| `google_merchant_center` | Google |
| `google_ads_account` | Google |
| `google_conversion_tag` | Google |
| `tiktok_ads_account` | TikTok |
| `tiktok_pixel` | TikTok |
| `tiktok_catalog` | TikTok |
| `snap_pixel` | Snapchat |
| `snap_catalog` | Snapchat |

Marketing Assets are **not** Sales Channels. They are marketing infrastructure.

### 3.4 Sales Channel

Sales Channels represent operational selling endpoints. Each belongs to exactly one Business Account.

**Examples:** Website, Facebook Shop, Instagram Shop, TikTok Shop, Amazon Egypt, Amazon KSA, POS, B2B Portal

**`channel_type` enum (required):**
| Value | Description |
|---|---|
| `website` | eCommerce website (WooCommerce, Shopify, custom) |
| `marketplace` | Third-party marketplace (Amazon, Noon, Jumia) |
| `social_commerce` | Social media store (Facebook Shop, Instagram Shop, TikTok Shop) |
| `pos` | Point of Sale terminal |
| `b2b` | B2B / wholesale portal |
| `custom` | Any non-standard channel type |

### 3.3 Warehouse

Belongs directly to Company. Never to Brand, Business Account, or Sales Channel. Shared inventory is an architectural requirement — a single warehouse may serve all brands and all business accounts of the company.

### 3.4 Product Ownership

Products belong only to Company. Never to Brand, Business Account, or Sales Channel. Publishing is handled separately by Commerce OS via Product Publications.

## 4. Product Publication (Commerce OS — Reserved by Organization)

Commerce OS will introduce the Product Publication model. Organization OS reserves the relationships but does not own this entity.

```
Product (Company-owned)
        ↓
Product Publication (Commerce OS)
        ↓
Business Account
        ↓
Sales Channel
```

Product Publication owns: Channel SKU, Channel Price, Sale Price, Publication Status, Visibility, Sync Status, Last Sync, Channel Images, Channel Description, External Product ID.

## 8. Invariants

1. Every Brand belongs to exactly one active Company.
2. Every Integration Account belongs to exactly one active Brand.
3. `integration_account.company_id` always equals `integration_account.brand.company_id`.
4. Every Sales Channel belongs to exactly one active Brand (direct ownership as of V2.3).
5. `channel.company_id` always equals `channel.brand.company_id`.
6. Every Marketing Account belongs to exactly one active Brand.
7. `marketing_account.company_id` always equals `marketing_account.brand.company_id`.
8. Every Marketing Asset belongs to exactly one Marketing Account.
9. A Warehouse belongs to a Company directly — never to a Brand or Integration Account.
10. Products belong to Company only — never published directly; publishing is handled via Product Publications (Commerce OS).
11. Marketing Accounts and Marketing Assets never own Orders.
12. Sales Channels never store advertising credentials, pixel IDs, or campaign data.
13. No new code may introduce a FK to `branches.id`.
14. Branch records must not be deleted during transition — soft-delete only.

---

## 9. Compliance Requirements for All Modules

- Commerce, Marketing, CRM, Procurement, Preparation, Logistics, Accounting, and the AI Platform must all conform to this model.
- No new development should introduce or depend on the legacy Branch concept.
- Any remaining Branch compatibility exists only as a temporary migration layer pending WP-ORG-008.
- Commerce modules must anchor order and product references to Sales Channel directly (not Integration Account).
- Marketing OS must anchor all campaign and attribution data to Marketing Account and Marketing Asset — never to Sales Channel directly.
- Attribution links Marketing Account → Sales Channel via UTM parameters / Order metadata only.
