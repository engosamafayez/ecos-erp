## 1. Identity Types

ECOS uses three identity types depending on the nature of the entity:

| Type | Format | When To Use |
|---|---|---|
| **UUID v4** | `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx` | Default for all aggregate roots and most entities |
| **ULID** | `01ARZ3NDEKTSV4RRFFQ69G5FAV` (26 chars) | High-volume append-only tables needing time-ordered IDs |
| **Business Number** | Human-readable sequence | Customer-facing identifiers (order numbers, invoice numbers) |

---

## 2. UUID (Default)

**UUID v4** is the default identity for all entities in ECOS.

**Why UUID:**
- Generated client-side (no database round-trip before insert)
- No sequential prediction (security benefit)
- Globally unique across distributed systems
- Standard support in PostgreSQL (native UUID type)

**When NOT to use UUID:**
- Very high-write append-only tables where index fragmentation matters (use ULID instead)
- Customer-facing reference numbers that humans must type or verify (use Business Number)

**Storage:** PostgreSQL `UUID` type (16 bytes native, not VARCHAR)

---

## 3. ULID (High-Volume Append-Only)

**ULID (Universally Unique Lexicographically Sortable Identifier)** is used for high-volume tables where time-ordering matters for query performance.

**ULID format:** 48-bit timestamp + 80-bit randomness = 128-bit total  
**Encoded as:** 26-character Crockford's Base32 string  
**Sortable:** Lexicographic sort = chronological sort

**Tables using ULID:**

| Table | Reason |
|---|---|
| `stock_movements` | High-write; queries order by occurred_at |
| `business_events` | Very high-write; ordered event processing |
| `receipt_layers` | FIFO consumption order must be chronological |
| `timeline_entries` | High-write; displayed in reverse chronological order |
| `notification_deliveries` | High-write; ordered by created_at |

**Storage:** PostgreSQL `CHAR(26)` or custom ULID type; indexed as B-tree (lexicographic = chronological)

**Benefit vs UUID:** New ULID values are always appended to the end of the B-tree index — no page splits, better write performance and index maintenance.

## 4. Business Numbers (Human-Readable)

Business numbers are human-readable identifiers generated by the application for customer-facing entities. They exist **alongside** the UUID primary key.

| Entity | Business Number Format | Example | Uniqueness |
|---|---|---|---|
| Order | `ORD-{year}{month}-{sequence}` | `ORD-202607-001234` | Per company |
| Invoice | `INV-{year}{month}-{sequence}` | `INV-202607-000089` | Per company |
| Purchase Order | `PO-{year}{month}-{sequence}` | `PO-202607-000012` | Per company |
| Supplier Return | `RTN-{year}{month}-{sequence}` | `RTN-202607-000003` | Per company |
| Customer | `CUST-{sequence}` | `CUST-000456` | Per company |
| Supplier | `SUP-{sequence}` | `SUP-000078` | Per company |

**Generation strategy:**
- Sequences are per-company, per-period (year+month for transactional entities)
- Sequence counter stored in a dedicated `sequences` table per (company_id, sequence_type, period)
- Generated atomically with the entity insert (no race condition)
- Business numbers are immutable once issued

**Business numbers are NEVER used as primary keys.** The UUID is always the PK and FK reference.

## 5. Identity Rules

| Rule | Statement |
|---|---|
| **ID-001** | UUID v4 is the default for all entities |
| **ID-002** | ULIDs are used only for explicitly listed high-volume append-only tables |
| **ID-003** | Business numbers are never primary keys; UUIDs are always the FK reference |
| **ID-004** | IDs are always generated application-side; never auto-increment or sequence at DB level |
| **ID-005** | IDs are immutable; once assigned, they never change |
| **ID-006** | External system IDs (WooCommerce, Meta) are stored in `external_reference` columns alongside ECOS UUIDs |
| **ID-007** | Cross-domain references use the referenced entity's UUID; no business number references across domains |

## 6. ID Generation in Application Layer

```
For UUID v4:
  PHP: Str::uuid() (Laravel helper) or Ramsey\Uuid\Uuid::uuid4()
  Type: UUID (stored as PostgreSQL UUID native type)

For ULID:
  PHP: Symfony\Component\Uid\Ulid::generate()
  Type: CHAR(26) (stored as fixed-length string)

For Business Numbers:
  Application service: BusinessNumberGenerator
    - Acquires lock on sequences row for (company_id, sequence_type, period)
    - Increments counter
    - Returns formatted business number
    - Releases lock
```
